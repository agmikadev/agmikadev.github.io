## Spacing Scale

Declared values (multiples of 4, confirmed from existing CSS):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tag hover offsets |
| sm | 8px | Analytics row gaps, alternatives gaps, CLI header gap |
| md | 16px | Card padding, analytics container padding, tab content padding |
| lg | 24px | Card-to-card gaps, tab header gap, analytics list gap |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#030305` | Background, void, base surfaces |
| Overlay surface | `rgba(10, 10, 20, 0.95)` + `backdrop-filter: blur(15px)` | Dashboard/BeltHUD overlay |
| Secondary (30%) | `rgba(255, 255, 255, 0.03)` (card bg), `rgba(255, 255, 255, 0.08)` (borders) | Cards, panels, dividers |
| Accent (10%) | Per-planet `--theme-color` CSS variable; Belt uses `#a9fc03` | Active tabs, progress fills, tag highlights, titles, glows |
| Skill proficiency | `#00f0ff` (≥90), `#00ff73` (≥75), `#ffae00` (≥50), `#ff3366` (<50) | AnalyticsTab bar colors — existing pattern |
| Destructive | `rgba(239, 68, 68, 0.8)` | Error states only (existing `.mission-error-log`) |

Accent reserved for:
1. Active tab indicator (border-bottom + text color + text-shadow glow)
2. Progress bar fill + box-shadow
3. Active mission tag background + border + glow
4. Planet/belt title text color + text-shadow
5. Planet debrief left border
6. BeltNetwork connection lines and node indicators
7. Custom scrollbar thumb in dashboard right panel

**Source:** Pre-populated from App.css (`--cp-bg-dark`, `--ai-glow`), PlanetaryDashboard.css, AnalyticsTab.css, MissionCard.css, Tabs.css, and PlanetaryData.ts (belt color `#a9fc03`).

## Interaction Contract

### Belt Click → BeltHUD Open

| Trigger | Action |
|---------|--------|
| User clicks belt SVG hitbox | `selectedPlanet` set to belt planet (`shape === "belt"`) |
| PlanetarySystem renders | Conditional: `selectedPlanet.shape === "belt"` → `<BeltHUD>`, else → `<PlanetDashboard>` |
| BeltHUD mounts | Overlay fades in (existing `fade-in` animation, 0.4s ease-out) |
| CliHeader completes | Content reveals with `slide-up-fade` animation (0.5s cubic-bezier) |

### BeltHUD Close

| Trigger | Action |
|---------|--------|
| User clicks "SYSTEM VIEW" button | `onClose()` callback → `setSelectedPlanet(null)` + `setHoveredPlanetId(null)` |
| BeltHUD unmounts | Overlay removed, planetary system visible again |

### Tab Switching

| Trigger | Action |
|---------|--------|
| User clicks `[ MISSÕES ]` or `[ MÉTRICAS ]` | `Tabs` component handles active state (existing pattern) |
| Tab content changes | `fade-in-view` animation (0.3s ease-out, existing pattern) |

### Planet Mission Content

All 5 planets already have `MissionsTab` and `AnalyticsTab` wired. Phase 3 ensures:
- Each planet's `tools` array has sufficient overlap with `MISSION_ARCHIVE` entries so missions display
- Each planet's `stats` array has accurate proficiency data (already populated in PlanetaryData.ts)
- No new planet content creation needed — data already exists
